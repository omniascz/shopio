## 0. Domain overview

### 0.1 Co tato doména **je**

- **Locales** — BCP-47 identifiers (`cs-CZ`, `sk-SK`, `en-US`, `de-DE`, `pl-PL`, `hu-HU`, `ro-RO`, `at-AT`)
- **Currencies** — ISO 4217 (CZK, EUR, USD, GBP, ...) with minor units, symbols
- **Exchange rates** — daily snapshots from ECB (EU), CNB (CZ), manual override option
- **Translations** — universal table for ANY entity field (product.title, category.name, cms_page.content_blocks, email_templates.subject, ...)
- **ICU MessageFormat** — plurals (Czech: 1 položka, 2-4 položky, 5+ položek), gender, ordinal
- **URL strategy** — path-based (`/cs/produkty/...`) OR domain-based (`shop.cz` vs `shop.de`) per `22-multistore-channels.md`
- **Hreflang** — cross-language alternate URLs (cross-ref `19`)
- **Locale auto-detect** — Accept-Language header → IP geo → customer preference → store default
- **Currency switching** — customer chooses; cart re-prices via exchange rates
- **Fallback chains** — `cs-CZ → cs → en → tenant.default`
- **Date/number/money formatting** — locale-aware via Intl.* APIs (server + client)
- **RTL support** — Arabic, Hebrew (Fáze 3+)
- **CJK support** — Chinese, Japanese, Korean fonts + vertical text (Fáze 4+)
- **Translation workflow** — admin UI per entity; bulk import/export (XLIFF, CSV); CAT tool integration
- **AI translation suggestions** — Anthropic/OpenAI auto-translate per-locale (Fáze 2+)
- **Locale-specific compliance** — RTL invoices, currency rounding (CZ rounds whole CZK), legal disclaimer texts

## 3. Data models

### 3.1 `locales` ([ENT-LOCALE-001](03-data-models-master.md#ent-locale-001))

Platform-wide reference table; per-tenant `enabled_locales` indicates which are active.

```sql
CREATE TABLE locales (
  code TEXT PRIMARY KEY,                                                                              -- BCP-47: 'cs-CZ', 'en-US', 'de-DE', 'zh-Hans-CN'
  language_code TEXT NOT NULL,                                                                        -- ISO 639: 'cs', 'en', 'de'
  region_code TEXT NULL,                                                                              -- ISO 3166-1 alpha-2: 'CZ', 'US', 'DE'
  script_code TEXT NULL,                                                                              -- ISO 15924: 'Latn', 'Cyrl', 'Hans', 'Arab'
  name_english TEXT NOT NULL,                                                                          -- "Czech (Czech Republic)"
  name_native TEXT NOT NULL,                                                                           -- "čeština (Česká republika)"
  direction TEXT NOT NULL CHECK (direction IN ('ltr','rtl')) DEFAULT 'ltr',
  decimal_separator TEXT NOT NULL DEFAULT '.',                                                          -- ',' for CZ/DE/EU
  thousands_separator TEXT NULL DEFAULT ',',                                                             -- ' ' for CZ, '.' for DE, ',' for US
  date_format TEXT NOT NULL DEFAULT 'YYYY-MM-DD',
  time_format TEXT NOT NULL DEFAULT 'HH:mm',
  first_day_of_week INTEGER NOT NULL DEFAULT 1,                                                           -- 0=Sunday, 1=Monday (ISO 8601)
  cldr_data JSONB NULL,                                                                                  -- subset of CLDR locale data (plural rules, currency format, ...)
  is_active BOOLEAN NOT NULL DEFAULT true,                                                                -- platform-level disable
  is_beta BOOLEAN NOT NULL DEFAULT false,                                                                 -- e.g., new languages
  fallback_locale_code TEXT NULL REFERENCES locales(code),                                                 -- 'cs-CZ' → 'cs' → 'en'
  -- audit
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Seed data (MVP active):** cs-CZ, sk-SK, en-US, en-GB, de-DE, de-AT, pl-PL, hu-HU, ro-RO, fr-FR (latter inactive by default).

### 3.2 `currencies` ([ENT-CURRENCY-001](03-data-models-master.md#ent-currency-001))

Platform-wide ISO 4217 reference.

```sql
CREATE TABLE currencies (
  code CHAR(3) PRIMARY KEY,                                                                              -- 'CZK', 'EUR', 'USD'
  numeric_code INTEGER NOT NULL,                                                                          -- ISO 4217 numeric
  name TEXT NOT NULL,                                                                                     -- "Czech Koruna"
  name_localized JSONB NULL,                                                                              -- per-locale name overrides

  minor_units INTEGER NOT NULL DEFAULT 2,                                                                  -- 2 for EUR/USD, 0 for JPY, 3 for BHD

  -- 'whole_unit' = round to whole currency unit (CZ tradition)
  -- 'five_cents' = round to 0.05 (CH tradition)
  -- 'banker' = banker's rounding
  -- 'swedish' = round to whole unit at total only (SE)
  is_active BOOLEAN NOT NULL DEFAULT true,
  is_crypto BOOLEAN NOT NULL DEFAULT false,                                                                -- Fáze 4+
  -- audit
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Seed:** ISO 4217 list. Active: CZK, EUR, USD, GBP, CHF, PLN, HUF, RON, SEK, NOK, DKK + ~20 majors.

### 3.5 `translations` ([ENT-TRANSLATION-001](03-data-models-master.md#ent-translation-001))

Universal translation table — uniform for any entity.

```sql
CREATE TABLE translations (
  id UUID PRIMARY KEY DEFAULT uuidv7(),
  tenant_id UUID NOT NULL REFERENCES tenants(id),
  entity_type TEXT NOT NULL,                                                                                -- 'product','category','collection','cms_page','blog_post','email_template','tax_class','shipping_method','custom_attribute'
  entity_id UUID NULL,                                                                                       -- NULL for global keys
  field TEXT NOT NULL,                                                                                       -- 'title','name','description_html','slug','seo_title','subject','body'
  locale TEXT NOT NULL REFERENCES locales(code),
  value TEXT NOT NULL,
  -- workflow
  status TEXT NOT NULL CHECK (status IN ('draft','published','outdated','pending_review','auto_translated','rejected')) DEFAULT 'published',
  source TEXT NOT NULL CHECK (source IN ('manual','import_csv','import_xliff','ai_anthropic','ai_openai','ai_deepl','crowdin','locize')) DEFAULT 'manual',
  source_value_at_creation TEXT NULL,                                                                         -- original value (default locale) when translation was created; detect outdating
  ai_confidence_score INTEGER NULL,                                                                            -- 0-100 (AI-generated translations)
  reviewed_at TIMESTAMPTZ NULL,
  reviewed_by_user_id UUID NULL,
  -- versioning
  previous_version_id UUID NULL REFERENCES translations(id),
  is_current BOOLEAN NOT NULL DEFAULT true,
  -- audit
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  created_by_user_id UUID NULL,
  metadata JSONB NOT NULL DEFAULT '{}'::jsonb,

  CONSTRAINT uq_translations_current UNIQUE (tenant_id, entity_type, entity_id, field, locale) DEFERRABLE INITIALLY IMMEDIATE
);

CREATE INDEX idx_translations_entity ON translations (entity_type, entity_id) WHERE is_current = true;
CREATE INDEX idx_translations_locale ON translations (tenant_id, locale, entity_type) WHERE is_current = true;
CREATE INDEX idx_translations_outdated ON translations (tenant_id, entity_type) WHERE status = 'outdated' OR status = 'pending_review';
CREATE INDEX idx_translations_text_gin ON translations USING GIN (to_tsvector('simple', value));            -- search translations
```

**Pattern:** uniqueness on (tenant, entity_type, entity_id, field, locale) for `is_current=true` rows. Old versions marked `is_current=false`.

## 4. Translation resolution

### 4.1 Resolution algorithm

Per request with `Accept-Language` header (e.g., `cs-CZ,cs;q=0.9,en;q=0.8`):

```
function resolveTranslation(entity_type, entity_id, field, requested_locale):
    # Try exact match
    t = SELECT value FROM translations
        WHERE tenant_id=$1 AND entity_type=$2 AND entity_id=$3
              AND field=$4 AND locale=$5 AND is_current=true
              AND status IN ('published','auto_translated')
    if t: return t.value

    # Fallback: language base (cs-CZ → cs)
    base = parseLocaleBase(requested_locale)
    if base != requested_locale:
        t = SELECT ... AND locale=$base
        if t: return t.value

    # Fallback: locale's fallback chain (per `locales.fallback_locale_code`)
    walk fallback chain (max 3 levels):
        t = SELECT ... AND locale=$current_fallback
        if t: return t.value

    # Fallback: tenant default locale
    t = SELECT ... AND locale=tenant.default_locale
    if t: return t.value

    # Last resort: return value from entity's master field (untranslated)
    return entity.field_master_value

    # If 'strict' fallback strategy: return NULL/empty instead
```

### 4.2 Best-match locale picker

Parse `Accept-Language` header (RFC 4647 best-match):
- `cs-CZ,cs;q=0.9,en;q=0.8` → preference: cs-CZ > cs > en
- Match against tenant's `enabled_locales`
- Fallback to `default_locale` if no match

Cached per (tenant, header_hash) for 5 min.

### 4.3 Bulk translation fetch

For listing pages (e.g., product list) — single query batched:

```sql
SELECT entity_id, field, value
FROM translations
WHERE tenant_id = $1 AND entity_type = 'product'
  AND entity_id = ANY($2)
  AND field = ANY($3)
  AND locale = $4 AND is_current = true;
```

Returns map for in-memory replacement in product objects.

### 4.4 Outdated translation detection

When master field changes:
1. `EVENT-{entity}-UPDATED` emitted (cross-ref `06`, `07`, ...)
2. `JOB-MARK-TRANSLATIONS-OUTDATED` runs
3. For each translation where `source_value_at_creation != current_master_value`: set `status='outdated'`
4. Admin UI flags translations needing re-review

### 4.5 ICU MessageFormat

For plural/gender-sensitive strings:

```icu
{count, plural,
   =0 {Žádné položky v košíku}
   one {# položka v košíku}
   few {# položky v košíku}
   other {# položek v košíku}}
```

Compiled at server-side render time (or client via formatjs). Czech plural rules: one (1), few (2-4), many (4+ for decimals), other (5+).

`translation_keys.is_icu=true` indicates ICU compilation needed.

## 5. Currency & exchange rates

### 5.1 Daily rate fetch

`JOB-FETCH-EXCHANGE-RATES-DAILY` at 14:30 UTC (after ECB publishes ~14:00):

```
For each tenant's exchange_rate_source:
   case 'ecb': fetch from https://www.ecb.europa.eu/stats/eurofxref/eurofxref-daily.xml
   case 'cnb': fetch from https://www.cnb.cz/cs/financni-trhy/devizovy-trh/kurzy-devizoveho-trhu/kurzy-devizoveho-trhu/denni_kurz.txt
   case 'currencyapi','fixer','openexchangerates': REST API call (paid subscription)
   case 'manual': skip

Parse and insert into exchange_rates table (idempotent via UNIQUE constraint).
```

**ECB:** rates published vs EUR base; we derive cross-rates (e.g., USD→CZK = (EUR→CZK) / (EUR→USD)).

**CNB:** rates published vs CZK base.

### 5.2 Currency conversion

For a money amount (X in currency A) → currency B:

```
rate = lookupRate(A, B, date)
if !rate:
    # Try inverse
    rate_inverse = lookupRate(B, A, date)
    if rate_inverse: rate = 1 / rate_inverse
    else:
        # Try via base (EUR or CZK)
        rate_a_to_base = lookupRate(A, base, date)
        rate_base_to_b = lookupRate(base, B, date)
        rate = rate_a_to_base * rate_base_to_b

converted = X * rate
apply rounding per currency B's rounding_strategy
apply tenant's exchange_rate_buffer_basis_points (e.g., +1% to absorb volatility for storefront display)
```

Cached by (A, B, date_truncated_hour).

### 5.3 Storefront currency switching

Customer selects EUR from dropdown:
1. Cart re-prices each item via exchange rate (per `11-cart.md` RULE-CART-009)
2. Display banner: "Prices shown in EUR (1 EUR = 24.50 CZK at ECB rate, 2026-05-20)"
3. Order placed in EUR; invoice in EUR
4. For CZ tax reporting: converted back to CZK at DUZP rate (per `15`)

### 5.4 Multi-currency invoicing

Invoice in cart currency (e.g., EUR). Per `15-tax-compliance.md`:
- VAT report values converted to tax authority currency (CZK for CZ-registered seller)
- Conversion at DUZP date rate from CNB (legal requirement)
- Recorded in `invoice.metadata.tax_authority_conversion`

### 5.5 Exchange rate buffer

For storefront pricing display, tenant can add buffer (`exchange_rate_buffer_basis_points`):
- Buffer 100 bp = +1% on top of mid-market rate
- Protects against intraday volatility
- Common practice for retail (banks add ~2-3%)

Different from CZ tax rate buffer (which uses exact CNB rate per law).

### 5.6 Manual rate override

Admin can set manual rate per (A, B, date):
- `source='manual'`
- Overrides automated daily fetch for that day
- Audited 100%
- Used pro:
  - Test environments
  - Special contractual rates
  - Holidays when feeds don't update

## 6. Locale routing

### 6.1 URL strategies

**Strategy A: Path-based** (default)
```
https://shop.example.com/cs/produkty/stolni-lampa-luna
https://shop.example.com/en/products/luna-table-lamp
https://shop.example.com/de/produkte/luna-tischlampe
```

**Strategy B: Domain-based** (per `22-multistore-channels.md`)
```
shop.cz → cs-CZ default
shop.de → de-DE default
shop.com → en-US default
```

Hybrid: domain-based stores + path-based within each.

### 6.2 Locale detection priority

```
1. URL path segment (/cs/, /en/) — if path-based store
2. Subdomain or domain (shop-de.example.com → de-DE) — if domain-based store
3. Customer.default_locale (if logged in)
4. Cookie: shopio_locale=cs-CZ
5. Accept-Language header (RFC 4647 best-match)
6. Geo-IP country → fallback locale (CZ IP → cs-CZ)
7. Store.default_locale
```

First match wins. Saved to cookie for sticky preference.

### 6.3 URL slugs per locale

Same entity, different slugs per locale:
- `cs-CZ`: `/cs/produkty/stolni-lampa-luna`
- `en-US`: `/en/products/luna-table-lamp`
- `de-DE`: `/de/produkte/luna-tischlampe`

Stored in `translations` with `field='slug'` per locale.

Resolver: `JOB-RESOLVE-URL` looks up entity by slug + locale.

Slugs unique per `(tenant_id, locale, parent_id, entity_type)` (per `06`, `07`).

### 6.4 Hreflang on storefront pages

Each page renders `<link rel="alternate" hreflang="...">` for all enabled locales of that entity (per `19-marketing-seo.md §4.6`).

`x-default` points to tenant default locale.

### 6.5 Locale switcher UI

Storefront component:
- Dropdown / flag list
- Shows current locale's `name_native`
- Click → redirects to same entity URL in selected locale (preserves context)
- Cookie set

## 7. State machines

### 7.1 Translation lifecycle

```
draft → published
        │
        ├→ outdated (master field changed)
        ├→ pending_review (auto-translated, awaiting human)
        └→ rejected (admin marked as bad translation)

auto_translated → reviewed → published
                          → rejected
```

### 7.2 Auto-translation request

```
queued → processing → completed → reviewed → applied (translation row updated)
                                           → rejected
                  └→ failed (provider error)
```

### 7.3 Translation import/export job

```
pending → processing → completed
                      → failed
```

## 8. Business rules

### RULE-I18N-001: Default locale always exists

Tenant.default_locale MUST be valid and in `enabled_locales`. Cannot disable last enabled locale.

### RULE-I18N-002: Fallback chain max 3 levels

To prevent infinite loops: locale fallback walk depth limited to 3. After: use tenant default.

### RULE-I18N-003: Slug uniqueness per locale

`(tenant_id, entity_type, parent_id, locale, slug)` UNIQUE for translatable slug fields.

Same product can have:
- `cs-CZ`: slug='stolni-lampa-luna'
- `en-US`: slug='luna-table-lamp'

But two products in same locale can't share slug.

### RULE-I18N-004: Translation versioning preserves history

When updating translation: old row marked `is_current=false`, new row inserted with `previous_version_id` link. Allows revert + audit trail.

Retention: 1 year of versions (after: prune to save space).

### RULE-I18N-005: Outdated translation detection

When master field changes (e.g., product.title in default locale), `JOB-MARK-TRANSLATIONS-OUTDATED` marks all language translations of that field as `status='outdated'`.

Stale translations still served (better than empty); admin flagged.

### RULE-I18N-006: ICU MessageFormat for plurals

Translation values may contain ICU syntax. Renderer compiles + interpolates.

If syntax error: log warning, render template literally (graceful degradation).

### RULE-I18N-007: Default locale doesn't need translation row

Master field value (e.g., `products.title`) is implicit default-locale translation. Save space by not duplicating.

Resolver checks for translation; if none in default locale: return master.

### RULE-I18N-008: Slug must be URL-safe per locale

Locale-specific characters allowed (e.g., German umlauts: `möbel`). URL-encoded if necessary. ASCII transliteration as alternative slug stored in metadata for fallback.

### RULE-I18N-009: Auto-translate requires review by default

`tenant_locale_settings.translation_review_required=true` (default): AI translations go through human review before publish.

Configurable per tenant (Fáze 2+: per-locale review policy).

### RULE-I18N-010: AI translation cost tracking

Each `auto_translation_requests` row records tokens + cost. Aggregated for billing transparency.

Tenant configurable budget cap (Fáze 2+).

### RULE-I18N-011: Translation memory matching

When translator works on entry, system suggests from `translation_memory`:
- Exact match: high confidence
- Fuzzy match (>70% similarity): suggestion with confidence score
- New text: no match; suggest AI translation if enabled

### RULE-I18N-012: Daily exchange rate freshness

`exchange_rates` for current date must exist:
- If missing: alert admin
- Fall back to most recent rate (max 7 days old)
- Stale > 7 days: critical alert, block multi-currency orders (configurable)

### RULE-I18N-013: CZ tenant uses CNB rates

CZ tax law mandates use of CNB exchange rates for accounting. Tenant default per country:
- CZ tenant: `exchange_rate_source='cnb'`
- Other EU: `exchange_rate_source='ecb'`
- Manual override always allowed

### RULE-I18N-014: Currency-specific rounding

Tax calculations apply per-currency rounding:
- CZK: optional whole-unit rounding (CZ tradition); recorded per `15-tax-compliance.md`
- EUR/USD: 2 decimal places (banker's rounding optional)
- JPY: 0 decimal places (whole yen)

### RULE-I18N-015: Locale fallback strict vs language

`tenant_locale_settings.fallback_strategy`:
- `strict`: only exact locale match served; missing translation = empty/null
- `language_fallback`: cs-CZ → cs → tenant default (recommended default)
- `default_locale`: missing locale → fallback directly to default

`strict` useful for legal docs where partial translation worse than none.

### RULE-I18N-016: Bulk import dry-run

`POST /translations:import-xliff?dry_run=true`:
- Validates file
- Reports rows would be created/updated
- Detects conflicts
- Doesn't apply changes

### RULE-I18N-017: Translation rate limits

Per `04 §13`. Specifically:
- Translation list `GET /translations`: 600/min admin
- Auto-translate request: 60/min per tenant (avoid runaway AI costs)
- Bulk import: 1/hour per tenant

### RULE-I18N-018: Email template per-locale fallback

Per `19-marketing-seo.md`:
- Email template for `cs-CZ` → fallback to `cs` → `en-US` → tenant default
- If recipient's locale missing template: use default locale + warning log

### RULE-I18N-019: Number format per locale

Use `Intl.NumberFormat` (server-side via Node, client-side via browser):
- cs-CZ: `1 234 567,89` (space thousands, comma decimal)
- de-DE: `1.234.567,89` (dot thousands, comma decimal)
- en-US: `1,234,567.89` (comma thousands, dot decimal)

### RULE-I18N-020: Date format per locale

Use `Intl.DateTimeFormat`:
- cs-CZ: `20. 05. 2026`
- en-US: `5/20/2026`
- de-DE: `20.05.2026`

ISO 8601 (`2026-05-20`) for machine readable APIs always.

### RULE-I18N-021: Money format per locale + currency

Combined:
- cs-CZ + CZK: `1 299,00 Kč`
- de-DE + EUR: `1.299,00 €`
- en-US + USD: `$1,299.00`

Currency symbol position per `currencies.symbol_position`.

### RULE-I18N-022: AI translation prompt template

Anthropic Claude / OpenAI prompt includes:
- Source language + target language
- Domain context (e-commerce product description)
- Tone (formal vs casual per tenant config)
- Brand voice notes (optional, from tenant config)
- Max output length (preserve UI constraints)
- Glossary terms (consistent brand vocabulary)

Per `33-ai-features.md` for prompt engineering.

### RULE-I18N-023: Storefront cookie locale

`shopio_locale` cookie:
- Set on locale switcher click
- HttpOnly: false (read by client JS)
- 1-year expiry
- SameSite=Lax

### RULE-I18N-024: RTL handling

Future Arabic/Hebrew support:
- `locales.direction='rtl'`
- Storefront CSS flips (logical properties: `margin-inline-start` etc.)
- Theme tested for RTL (Fáze 3+)

### RULE-I18N-025: CJK handling

Future Chinese/Japanese/Korean:
- `locales.script_code='Hans'/'Hant'/'Jpan'/'Hang'`
- Web font preloading for performance
- Vertical text layout option (Fáze 4+)

### RULE-I18N-026: Translation review SLA

Auto-translated entries require review within configurable window (default 7 days). After SLA: admin alert + can be auto-published with `metadata.auto_published_after_sla=true`.

### RULE-I18N-027: Cross-locale slug redirect

If product slug changes in one locale, prior slug → redirect 301 (per `19-marketing-seo.md`).

E.g., old slug `stara-lampa` → new `stolni-lampa-luna` → redirect.

### RULE-I18N-028: Locale-specific compliance text

Legal disclaimers per locale (cookie banner, T&C link, GDPR notice). Stored in `translation_keys` with `namespace='legal'`.

Per CZ law: stronger disclosure for distance contracts; per DE: stricter ToS templates.

### RULE-I18N-029: Translation export formats

- **XLIFF 2.0** — industry standard CAT tools (Trados, MemoQ, Crowdin)
- **CSV** — simple spreadsheet workflow
- **JSON** — developer-friendly
- **PO** (gettext) — legacy / Linux tools
- **XLSX** — non-technical translators

```typescript
async function fetchExchangeRatesDaily() {
  const today = new Date();

  // ECB
  const ecbRates = await fetchEcbRates(today);  // XML feed parse
  await pg.execute(sql`
    INSERT INTO exchange_rates (base_currency, quote_currency, rate, valid_for_date, source)
    VALUES ${ecbRates.map(r => `('EUR', '${r.quote}', ${r.rate}, '${today}', 'ecb')`).join(',')}
    ON CONFLICT (base_currency, quote_currency, valid_for_date, source) DO UPDATE SET rate = EXCLUDED.rate, fetched_at = now()
  `);

  // CNB
  const cnbRates = await fetchCnbRates(today);
  await pg.execute(sql`...`);  // similar pattern

  // Emit event
  await emitOutbox('EVENT-EXCHANGE-RATES-FETCHED', { source: 'ecb,cnb', count: ecbRates.length + cnbRates.length, date: today });
}
```

### 12.2 JOB-EXECUTE-AUTO-TRANSLATION detail

```typescript
async function executeAutoTranslation(requestId: string) {
  const req = await loadAutoTranslationRequest(requestId);
  await updateStatus(req, 'processing');

  const provider = providers[req.provider];
  const prompt = buildPrompt(req);
  
  try {
    const result = await provider.translate(req.source_text, req.source_locale, req.target_locale, prompt);
    
    await pg.execute(sql`
      UPDATE auto_translation_requests
      SET status='completed', result_text=${result.text}, confidence_score=${result.confidence},
          tokens_consumed=${result.tokens}, cost_amount=${result.cost},
          completed_at=now()
      WHERE id=${requestId}
    `);
    
    await emitOutbox('EVENT-AUTO-TRANSLATION-COMPLETED', { request_id: requestId, target_locale: req.target_locale, confidence: result.confidence });

    // If review not required: auto-apply
    const settings = await loadTenantLocaleSettings(req.tenant_id);
    if (!settings.translation_review_required) {
      await applyAutoTranslation(req, result);
    }
  } catch (err) {
    await updateStatus(req, 'failed', err.message);
  }
}
```

## 14. Edge cases & error handling

| Scenario | Behavior | Code |
|---|---|---|
| Disable last enabled locale | Reject | `MUST_HAVE_ONE_LOCALE`, 422 |
| Disable default locale | Reject; choose new default first | `CANNOT_DISABLE_DEFAULT_LOCALE`, 422 |
| Locale code invalid | Reject (validates against BCP-47 patterns) | `INVALID_LOCALE_CODE`, 422 |
| Currency code invalid | Reject (ISO 4217) | `INVALID_CURRENCY_CODE`, 422 |
| Exchange rate missing for required conversion | Use latest available + warn (or block if > 7 days old, per RULE-I18N-012) | (handled) |
| Translation ICU syntax error | Log warning; render literally | (graceful) |
| Auto-translate provider down | Retry; fallback provider per priority order | (handled) |
| Auto-translate result low confidence | Mark `pending_review`; never auto-apply if `review_required=true` | (handled) |
| Translation conflict on import | Mark `status='rejected'` with reason; admin resolves | (handled) |
| URL slug collision with deleted entity's old slug | Redirect old slug to new (per `19`) | (handled) |
| Locale fallback chain reaches cycle | Detect + truncate at 3 levels | (handled per RULE-I18N-002) |
| Customer chooses locale not enabled by tenant | Reject; fall back to closest match | `LOCALE_NOT_ENABLED`, 422 |
| Exchange rate stale > 7 days | Block multi-currency orders (configurable); alert admin | (handled per RULE-I18N-012) |
| AI translation cost budget exceeded | Block new requests; admin alert | (Fáze 2+) |
| Bulk import 50k rows | Stream processing; chunked commits | (handled) |
| Translation memory false positive | Translator overrides; system learns from edits (Fáze 2+) | (handled) |
| RTL content in LTR template | Theme renders correctly (logical CSS); Fáze 3+ | (Fáze 3+) |
| Multi-currency invoice round error | Use deterministic banker's rounding; document in metadata | (handled) |
| Exchange rate buffer creates customer-visible mismatch with bank charge | Disclosed in checkout: "Final charged amount may vary due to bank's rate" | (handled) |
| Concurrent translation edits | Optimistic lock via version | `RESOURCE_VERSION_MISMATCH`, 412 |
| Locale support claims (e.g., "fr-FR" enabled but no products translated) | Storefront shows "Translation in progress" badge; falls back to default | (handled) |

## 15. Performance

### 15.1 Targets

| Operation | p50 | p95 | p99 |
|---|---|---|---|
| Single translation lookup | 1 ms | 3 ms | 10 ms |
| Bulk translation fetch (50 entities × 5 fields × 1 locale) | 10 ms | 30 ms | 80 ms |
| `GET /storefront/translations/{ns}/{locale}` (cached) | 5 ms | 15 ms | 40 ms |
| Exchange rate lookup | 2 ms | 5 ms | 15 ms |
| Currency conversion | 1 ms | 3 ms | 10 ms |
| Locale resolution (Accept-Language parse) | 2 ms | 5 ms | 15 ms |
| Auto-translate request (Anthropic 200 tokens) | 1500 ms | 4000 ms | 10000 ms |
| Bulk export XLIFF (10k strings) | 5 s | 20 s | 60 s |
| `JOB-FETCH-EXCHANGE-RATES-DAILY` | 10 s | 30 s | 60 s |

### 15.2 Optimization

- **Translation cache** Redis per (tenant, entity_type, locale, batch_hash) 5 min TTL
- **Exchange rate cache** Redis per (base, quote, hour) 1 hour TTL
- **Storefront UI strings cache** CDN-friendly (Cache-Control + Vary: Accept-Language)
- **Locale resolution cache** per Accept-Language hash 5 min
- **Materialized view** for product display (joins translations into product table, refreshed hourly)
- **DataLoader v GraphQL** for batched translation fetches

### 15.3 Hot path

```sql
-- Bulk fetch for product list
SELECT entity_id, field, value
FROM translations
WHERE tenant_id = $1 AND entity_type = 'product'
  AND entity_id = ANY($2)
  AND field = ANY($3)
  AND locale = $4 AND is_current = true AND status = 'published';
-- Uses idx_translations_entity + locale composite
```

```sql
-- Exchange rate lookup
SELECT rate FROM exchange_rates
WHERE base_currency = $1 AND quote_currency = $2
  AND valid_for_date <= $3
ORDER BY valid_for_date DESC, source ASC
LIMIT 1;
-- Uses idx_exchange_rates_lookup
```

### 15.4 Scaling

- Translation table grows linearly with catalog × locales. 100k products × 5 locales × 5 fields = 2.5M rows manageable
- Beyond: partition translations by tenant_id (Fáze 3+)
- Translation cache hit rate target: >95%
